What is an LLM?

  • Token prediction systems
  • Trained on public knowledge
  • Predictable but non-deterministic by default

Api controls

  • Randomness — temprature, top_t (predictable <> unpredictable)
  • Response Format — Output management
  • Tools — how much text fits in one request

Harness Engineering

  • Prompt Engineering (instructions)
  • Context Engineering (input knowledge including tools)
  • Flow control (behaviour)

Response Format

Structured output

  • JSON / schema-shaped
  • Easy to validate
  • Deterministic shape
  • Safe to turn into app actions

Tool calling

User request
↓
LLM chooses a tool + arguments
↓
Your app runs the tool
↓
UI updates / result returns

The model decides what to do. Your code decides how it happens.

Evaluation and predictability

  • Keep the tool set small
  • Models tend get confused with conflicting/too much context
  • Use evaluation tests to check important behaviors
Good AI features are not just smart — they are constrained and testable.
