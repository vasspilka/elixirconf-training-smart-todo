Class 3 — Building LLM-Powered Applications

ElixirConf EU 2026

LLMs are stateless

  • The model does not remember previous calls
  • Every request is independent
  • Conversation memory is something we build
“Memory” is just the message history you send back each time.

Chat = a message list

  • System — rules and context
  • User — what the human said
  • Assistant — previous model replies

On each turn, you resend the conversation state.

Context is a budget

  • Conversation history consumes tokens
  • You cannot keep everything forever
  • Keep the important context, not all context
Good chat systems choose what the model should remember.

RAG keeps answers grounded

User asks a question
↓
Retrieve relevant cards
↓
Add them to the prompt
↓
Generate an answer based on real board data

This is where embeddings become useful in chat.

Streaming improves UX

  • Without streaming: wait for the whole answer
  • With streaming: show the answer as it is generated
  • Same model, better user experience
Streaming makes the assistant feel responsive and alive.

What we are building now

  • A board-aware chat assistant
  • Relevant board context on each turn
  • Streaming responses in the UI

Let’s build it.
