When one call is not enough

  • Some tasks need multiple steps
  • Example: research a topic, compare findings, then create tasks
  • That is where agents start to help

What is an agent?

An agent is an LLM in a loop:
plan → act → observe → repeat

Same tool-calling pattern as before, just repeated.

Research agent flow

Goal from the user
↓
Inspect the current board
↓
Search for relevant information
↓
Propose cards or updates
↓
User reviews the result

Keep it bounded

  • Use a small tool set
  • Limit the number of steps
  • Prefer review before making many changes
Agents are powerful, but they are also slower, more expensive, and less predictable.

What we are building now

  • A simple research agent for our board
  • Research first, then propose task cards
  • Keep the workflow understandable and reviewable
